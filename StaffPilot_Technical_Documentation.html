StaffPilot Technical Documentation

Executive Summary

StaffPilot is an AI-powered marketing automation platform that enables businesses to create, schedule, and publish content across multiple social media platforms. The platform leverages advanced AI for content generation, image creation, and video production, with integrations to major advertising platforms.

2. Social Media Posting Implementation

2.1 LinkedIn Posting ✅ Active

How it works:

  1. Authentication: OAuth 2.0 flow via LinkedIn's authorization dialog
  2. Token Storage: Access tokens stored securely with automatic refresh
  3. Post Creation: Uses LinkedIn UGC (User Generated Content) API v2
  4. Media Upload: Supports images and videos via LinkedIn's media upload endpoint

Key Features: Personal profile and Company Page posting, Image and video attachments, Automatic token refresh

2.2 Twitter/X Posting ✅ Active

How it works:

  1. Authentication: Dual OAuth system
    • OAuth 2.0: For tweet creation (login, posting text)
    • OAuth 1.0a: Required for media uploads (Twitter API limitation)
  2. Token Management: Automatic refresh with expiration tracking
  3. Post Creation: Twitter API v2
  4. Media Upload: Twitter Media Upload API v1.1 with chunked upload for large files

Key Features: Text tweets with images/videos, Automatic token refresh, Chunked upload for videos > 5MB

2.3 Facebook Posting

⚠️ Status: Pending Meta Developer Setup - Not active in production yet

Authentication: OAuth 2.0 via Facebook Login dialog

API: Facebook Graph API v18.0 for Page posts (text, photos, videos)

2.4 Instagram Posting

⚠️ Status: Pending Meta Developer Setup - Not active in production yet

Authentication: Via Facebook OAuth (Instagram Business accounts linked to Facebook Pages)

Post Creation: Two-step process using Instagram Content Publishing API

2.5 TikTok Posting ✅ Active

Authentication: TikTok OAuth 2.0 with PKCE

Features: Video upload via TikTok Content Posting API with chunked upload

3. Content Generation (AI)

Note: Currently only Google Gemini is active in production. OpenAI and Anthropic integrations are implemented and ready but not enabled.

3.2 Content Generation Flow

  1. User provides an optional prompt/request for additional information
  2. System uses RAG (Retrieval Augmented Generation) with tenant's documents
  3. LLM generates content tailored to brand voice and specific social media platform
  4. Content optionally enhanced with AI-generated images/videos
  5. Published to selected platforms

3.3 Image Generation

Provider: Google Gemini with Imagen 4.0 model

Features: Brand asset references for consistency, Multiple aspect ratios, Batch generation

3.4 Video Generation

Provider: Google Gemini with Veo 2 model

Features: 8-60 second videos, Video extension for longer content, Brand asset integration

4. Campaign Creation

AI-Powered Campaign Planning:

  1. User provides campaign objectives, target audience, channels, budget
  2. AI researches market using:
    • RAG: Searches tenant's uploaded documents
    • SERP API: Real-time market/competitor research
  3. AI generates structured campaign plan with timeline, budget allocation, ad sets
  4. Plan presented for user approval

Asset Generation:

  • Headlines (short: 30 chars, long: 90 chars)
  • Descriptions (90 chars)
  • Images (various dimensions)
  • Videos (8-60 seconds)

7. OAuth Integrations

For Social Media Posting: Each user connects their own accounts via OAuth. The platform stores access tokens (encrypted), refresh tokens, token expiration times, and selected pages/profiles.

For Deployment: Railway (backend services), Vercel (frontend), GitHub (code repository)
